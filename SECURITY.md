# Security Policy

Thank you for helping keep DJConnect safe.

## Reporting a Vulnerability

Please report security issues privately by emailing:

```text
security@djconnect.dev
```

Do not open a public GitHub issue for suspected vulnerabilities, leaked secrets,
private Home Assistant URLs, tokens, diagnostics, or personal data.

When reporting, include enough detail to help reproduce or understand the
issue:

- affected platform: iOS, iPadOS, macOS, Home Assistant integration, release
  tooling, or documentation;
- affected version, commit, or release tag if known;
- clear reproduction steps or a proof of concept;
- expected impact;
- logs or screenshots only after removing tokens, private URLs, credentials,
  pairing codes, and personal data.

Please do not include Spotify OAuth credentials, Home Assistant long-lived
access tokens, DJConnect bearer tokens, WiFi passwords, private network URLs,
or raw diagnostics containing sensitive values.

## Response

The maintainer will review reports as soon as reasonably possible, may ask for
clarifying details, and will coordinate any fix, release, or disclosure timing
privately before public discussion.

## Supported Versions

Security fixes are normally made against the latest released DJConnect app
version and the current `main` branch. Older releases may receive fixes when
the issue is severe and a backport is practical.
