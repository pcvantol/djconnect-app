# DJConnect App 3.2.17

## Changed

- DJConnect now calls the canonical Home Assistant API routes under
  `/api/djconnect/v1/...`.
- Pairing QR codes and deep links now advertise
  `pair_path=/api/djconnect/v1/pair`.
- API and release documentation now describe the v1 route prefix throughout.

## Fixed

- Added regression coverage to keep Apple client HTTP routes on the v1 prefix.
- Server-provided image, TTS, and audio URLs remain untouched by the client.
- VibeCast artist artwork decoding is covered for proxied
  `context.artist_image_url` values, with external image URLs ignored.
- Discover no longer replaces the page with a load-error card; users stay on
  the empty or locked state and can retry from the page controls.
