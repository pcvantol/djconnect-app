# DJConnect App 3.2.8

- DJConnect now has a shared localization foundation for English, Dutch,
  German, French and Spanish across the Apple clients.
- Pairing and Home Assistant API errors now use one central user-facing message
  mapper, including wrong app type, invalid pair code, stale auth and
  not-configured responses.
- Protocol values such as `client_type`, endpoint paths and JSON keys stay
  unchanged and are not localized.
- A localization validator now checks that every shared key exists for all five
  supported languages and keeps format placeholders aligned.
- Contributor guidance now requires user-facing Apple client text to be
  localized for every supported language in the same pull request.
