# DJConnect App 3.2.28

## Gewijzigd

- Pushregistratie volgt nu het Central API v1.0.11 bootstrapcontract:
  Apple-clients halen kortlevende `djcboot_...` proofs op via een vertrouwde
  pairing issuer, nooit rechtstreeks via Home Assistant.
- Apple-clients bewaren nu `ha_install_id`, `integration_version` en optioneel
  `pairing_session_id` uit Home Assistant pairing/status responses en sturen
  die metadata mee naar Central zonder APNs-token.

## Opgelost

- Push-bootstrap-herstel probeert Home Assistant nu exact één keer opnieuw,
  bewaart proofs alleen in memory, valideert de issuer-expiry en voorkomt
  retry-loops bij ongeldige of verlopen proofs.
- Central bootstrap-fouten zoals `invalid_client_type`,
  `invalid_app_bundle_id`, `invalid_push_environment` en
  `bootstrap_rate_limited` worden nu als privacy-veilige statuscodes bewaard.
