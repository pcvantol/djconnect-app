# DJConnect App 3.2.26

## Changed

- Playback refreshes and backend recovery retries now coalesce better, reducing
  repeated network work after commands.
- The local issue and TODO documentation now reflects the completed hardening
  work.
- iOS runtime fixture coverage now includes pairing, runtime state, queue,
  playlists, Ask DJ, voice unavailable, and the Now Playing favorite button.

## Fixed

- Temporary `backend_unavailable` outages now show recovery copy, retry
  automatically, and keep pairing intact.
- Generic playback-backend unavailable command failures now use the recovery
  copy while specific playback restrictions still show their own message.
- Voice/audio loading, incoming payload limits, runtime log redaction, and
  duplicate queue row identity handling are hardened.
- Apple Watch pairing through the iPhone proxy now keeps a valid Watch QR
  pending until the Watch registration is available, so the Home Assistant pair
  request starts even when WatchConnectivity activation arrives just after the
  scan.
- VibeCast AirPlay videos now preload Track Insight or Now Playing artwork
  before encoding, so the AirPlay stream no longer bakes in the placeholder
  image while the live VibeCast window loads album art.
